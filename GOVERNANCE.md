# Governance

## Project Model

Buildev follows a maintainer-led model.

Maintainers review pull requests, triage issues, and make release decisions.

## Decision Process

1. Open proposal in an Issue or Discussion.
2. Gather feedback from contributors.
3. Maintainers decide and document rationale.

For major architecture changes, include:

- problem statement
- alternatives considered
- migration impact
- test plan

## Roles

- **Maintainers**: merge PRs, manage releases, resolve conflicts
- **Contributors**: submit PRs, improve docs, report bugs, review

## Release Expectations

- Keep changelog entries clear.
- Follow Conventional Commits.
- Ensure lint, typecheck, and tests pass before release.
