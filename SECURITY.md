# Security Policy

## Supported Versions

Security fixes are provided for the latest `main` branch and the latest release line.

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

Instead:

1. Open a private vulnerability report on GitHub Security Advisories, or
2. Contact maintainers directly with reproduction steps and impact.

Include:

- affected area
- exact steps to reproduce
- proof of concept if available
- severity and potential impact

## Response Targets

- Initial acknowledgment: within 72 hours
- Triage decision: within 7 days
- Fix or mitigation plan: as soon as validated

## Disclosure

After a fix is available, maintainers will coordinate responsible disclosure and publish release notes.
